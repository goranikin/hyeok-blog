## The Architecture

![archi](/writing/smart-home-2/archi.png)

The data flow of sensor example

1. Sensor broadcasts temperature and humidity over Zigbee radio (Sensor -> SONOFF Dongle)
2. Raw Zigbee data flows through USB cable to the Zigbee2MQTT container (SONOFF Dongle -> Zigbee2MQTT)
3. Translated into MQTT message "zigbee2mqtt/temp_sensor" → \{"temperature": 23.5, "humidity": 45\} (Zigbee2MQTT -> Mosquitto)
4. HA subscribes to Zigbee2MQTT/# topics and receives the sonsor data automatically (Mosquitto -> Home Assistant)
5. I can see "My Room: 23.5°C, 45% humidity" on my HA dashborad!

I can also control through the reverse flow.

MQTT stands for Message Queueing Telemetry Transport. It's designed for IoT senarios where you need lightweight and reliable
messaging between many small devices. I think of it as a kind of communication protocol, like HTTP.

Mosquitto is the MQTT broker -- it receives message from publishers and delivers them to subscribers. Zigbee2MQTT is the
translator between Zigbee radio and MQTT messages. I can build intricate scripts to interact with smart devices and integrate them
via Mosquitto down the line.

If you want a simpler archtecture, you can use ZHA, which connects the Zigbee dongle directly on HA.

## Specifications

I use docker compose to run three containers via an internal docker network.
1. Home Assistant
2. Mosquitto
3. Zigbee2MQTT

I provide the specific configuration files in the last section of this post, but you will need to adjust the settings based on your own devices.
After running `docker compose up`, I can access the HA dashboard at `http://<mini PC's IP>:<HA Port>`.

## Appendix

docker compose
```docker
services:
  # ─── Home Assistant ───────────────────────────────────────
  homeassistant:
    container_name: homeassistant
    image: ghcr.io/home-assistant/home-assistant:stable
    restart: unless-stopped
    ports:
      - "oooo:oooo"
    volumes:
      - ./homeassistant:/config
      - /etc/localtime:/etc/localtime:ro
    environment:
      - TZ=Asia/Seoul
    depends_on:
      - mosquitto
    networks:
      - ha_network

  # ─── Mosquitto MQTT Broker ────────────────────────────────
  mosquitto:
    container_name: mosquitto
    image: eclipse-mosquitto:2
    restart: unless-stopped
    ports:
      - "oooo:oooo"
      - "oooo:oooo"
    volumes:
      - ./mosquitto/config:/mosquitto/config
      - ./mosquitto/data:/mosquitto/data
      - ./mosquitto/log:/mosquitto/log
    networks:
      - ha_network

  # ─── Zigbee2MQTT ──────────────────────────────────────────
  zigbee2mqtt:
    container_name: zigbee2mqtt
    image: ghcr.io/koenkk/zigbee2mqtt
    restart: unless-stopped
    ports:
      - "oooo:oooo"
    volumes:
      - ./zigbee2mqtt:/app/data
      - /run/udev:/run/udev:ro
    devices:
      - /dev/serial/by-id/<my_zigbee_gateway_serial_number>:/dev/ttyUSB0
    environment:
      - TZ=Asia/Seoul
    depends_on:
      - mosquitto
    networks:
      - ha_network

networks:
  ha_network:
    driver: bridge
```
